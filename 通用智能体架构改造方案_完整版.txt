一、核心设计理念转变
================================================================================

❌ 旧理念（场景驱动）：
   "如果是财报任务，创建 researcher+analyzer+writer"
   "如果是旅行任务，创建 flight_finder+hotel_booker+itinerary_planner"
   → 无限分类，无法泛化，每新场景都要开发新智能体

✅ 新理念（能力驱动）：
   "只有一个通用智能体"
   "所有能力都通过工具/插件提供"
   "智能体只负责：理解目标 → 规划步骤 → 调用工具 → 整合结果"
   → 一套架构，适应所有场景，新能力只需接入新工具

核心类比：
   旧系统 = 瑞士军刀（每个场景一个专用刀片）
   新系统 = 智能手机（一个通用设备，能力通过 APP 扩展）

  + class UniversalAgent:
    - async def think(self, observation: str) -> Thought:
        # 调用 LLM 决定下一步
        prompt = build_thought_prompt(observation, self.available_tools)
        return await llm.generate_json(prompt)
    
    - async def act(self, thought: Thought) -> Observation:
        # 执行动作（调用工具或结束）
        if thought.action == "call_tool":
            return await self.call_tool(thought.tool, thought.args)
        elif thought.action == "finish":
            return await self.synthesize_answer()
    
    - async def reflect(self, observation: Observation) -> Reflection:
        # 评估执行质量，决定是否继续
        quality_score = await self.evaluate_quality(observation)
        return Reflection(should_continue=quality_score > threshold)
    
    - async def execute(self, goal: str) -> AsyncGenerator[Event, None]:
        # ReAct 循环
        state = ExecutionState(goal=goal)
        while not state.is_completed and state.current_step < max_steps:
            observation = build_observation(state)
            thought = await self.think(observation)
            yield Event(type="supervisor_thought", data=thought)
            
            if thought.action == "call_tool":
                yield Event(type="tool_call_start", data=thought.tool)
                result = await self.call_tool(thought.tool, thought.args)
                yield Event(type="tool_call_end", data=result)
                
                reflection = await self.reflect(result)
                yield Event(type="reflection", data=reflection)
                
                state.history.append((thought, result))
                state.current_step += 1
                
            elif thought.action == "finish":
                async for chunk in self.synthesize_answer(state):
                    yield Event(type="final_answer_chunk", data=chunk)
                state.is_completed = True

     ```python
         def __init__(self):

             return await client.call_tool(tool_name, args)
     ```

  + class ReactEngine:
      def __init__(self, llm_provider, tool_manager):
          self.llm = llm_provider
          self.tools = tool_manager
          self.max_steps = 20
          self.step_timeout = 60  # 秒
      
      async def execute(self, goal: str) -> AsyncGenerator[dict, None]:
          """ReAct 循环执行器"""
          history = []
          step = 0
          
          # Step 0: 初始规划
          plan = await self.generate_plan(goal)
          yield {"type": "task_plan", "data": plan}
          
          while step < self.max_steps:
              # Think
              thought = await self.think(goal, history)
              yield {"type": "supervisor_thought", "data": thought}
              
              # Act
              if thought["action"] == "call_tool":
                  tool_result = await self.execute_tool(
                      thought["tool"], 
                      thought["args"],
                      timeout=self.step_timeout
                  )
                  yield {"type": "tool_call_end", "data": tool_result}
                  history.append({"thought": thought, "result": tool_result})

                  reflection = await self.reflect(goal, history)
                  yield {"type": "reflection", "data": reflection}
                  
                  if reflection["should_finish"]:
                      break
                      
              elif thought["action"] == "finish":
                  # Synthesize
                  async for chunk in self.synthesize(goal, history):
                      yield {"type": "final_answer_chunk", "data": chunk}
                  break
              
              step += 1
          
          yield {"type": "done", "data": {"steps": step, "history": history}}

【修改 1】创建执行状态管理 Store

```javascript
import { create } from 'zustand';

const executionStore = create((set, get) => ({
  // 当前执行状态
  currentExecution: null,  // { id, goal, status, startTime }
  
  // 执行历史（用于恢复）
  executionHistory: [],
  
  // 结构化数据
  supervisorThoughts: [],
  taskPlan: null,
  toolCalls: [],
  reflections: [],
  finalAnswer: '',
  
  // 状态更新方法
  startExecution: (goal) => set({
    currentExecution: {
      id: Date.now(),
      goal,
      status: 'running',
      startTime: new Date().toISOString()
    },
    supervisorThoughts: [],
    taskPlan: null,
    toolCalls: [],
    reflections: [],
    finalAnswer: ''
  }),
  
  addThought: (thought) => set((state) => ({
    supervisorThoughts: [...state.supervisorThoughts, thought]
  })),
  
  updateTaskPlan: (plan) => set({ taskPlan: plan }),
  
  addToolCall: (toolCall) => set((state) => ({
    toolCalls: [...state.toolCalls, toolCall]
  })),
  
  updateToolCallProgress: (toolName, progress) => set((state) => ({
    toolCalls: state.toolCalls.map(tc =>
      tc.tool === toolName ? { ...tc, ...progress } : tc
    )
  })),
  
  addReflection: (reflection) => set((state) => ({
    reflections: [...state.reflections, reflection]
  })),
  
  appendAnswer: (chunk) => set((state) => ({
    finalAnswer: state.finalAnswer + chunk
  })),
  
  completeExecution: (stats) => set((state) => ({
    currentExecution: { ...state.currentExecution, status: 'completed', stats },
    executionHistory: [...state.executionHistory, state.currentExecution]
  }))
}));

export default executionStore;
```

【修改 2】创建 SSE 事件处理 Hook

新建文件：/workspace/frontend/src/hooks/useAgentExecution.js

```javascript
import { useEffect, useCallback } from 'react';
import executionStore from '../store/executionStore';
import { chatAPI } from '../services/api';

export function useAgentExecution(sessionId) {
  const store = executionStore();
  
  const handleSSEEvent = useCallback((event) => {
    console.log('[useAgentExecution] 收到事件:', event.type, event.data);
    
    switch (event.type) {
      case 'session_info':
        // 初始化会话
        break;
        
      case 'supervisor_thought':
        store.addThought(event.data);
        break;
        
      case 'task_plan':
        store.updateTaskPlan(event.data);
        break;
        
      case 'tool_call_start':
        store.addToolCall({
          tool: event.data.tool,
          status: 'running',
          startTime: Date.now(),
          params: event.data.params
        });
        break;
        
      case 'tool_call_progress':
        store.updateToolCallProgress(
          event.data.tool,
          event.data
        );
        break;
        
      case 'tool_call_end':
        // 更新工具调用状态
        store.addToolCall({
          tool: event.data.tool,
          status: event.data.status,
          result: event.data.result,
          endTime: Date.now()
        });
        break;
        
      case 'reflection':
        store.addReflection(event.data);
        break;
        
      case 'final_answer_chunk':
        store.appendAnswer(event.data.chunk);
        break;
        
      case 'error':
        console.error('[Execution Error]', event.data);
        break;
        
      case 'done':
        store.completeExecution(event.data);
        break;
        
      default:
        console.warn('未知事件类型:', event.type);
    }
  }, [store]);
  
  const execute = useCallback(async (query) => {
    store.startExecution(query);
    
    try {
      await chatAPI.sendChatStream(
        query,
        sessionId,
        null,
        handleSSEEvent
      );
    } catch (error) {
      console.error('[Execution Failed]', error);
      store.completeExecution({ error: error.message });
    }
  }, [sessionId, handleSSEEvent, store]);
  
  return {
    execute,
    thoughts: store.supervisorThoughts,
    taskPlan: store.taskPlan,
    toolCalls: store.toolCalls,
    reflections: store.reflections,
    finalAnswer: store.finalAnswer,
    status: store.currentExecution?.status || 'idle'
  };
}
```

优先级：P0

```javascript
class SSEParser {
  constructor(onEvent, onError) {
    this.buffer = '';
    this.eventQueue = [];
    this.lastEventTime = Date.now();
    this.onEvent = onEvent;
    this.onError = onError;
    this.heartbeatInterval = null;
  }
  
  startHeartbeat() {
    this.heartbeatInterval = setInterval(() => {
      if (Date.now() - this.lastEventTime > 30000) {
        console.warn('[SSE] 30 秒无事件，可能已断线');
        // 触发重连逻辑
      }
    }, 10000);
  }
  
  parse(chunk) {
    this.lastEventTime = Date.now();
    this.buffer += chunk;
    
    const lines = this.buffer.split('\n');
    this.buffer = lines.pop();
    
    for (const line of lines) {
      if (line.startsWith('event: ')) {
        this.currentEvent = { type: line.slice(7) };
      } else if (line.startsWith('data: ')) {
        const data = JSON.parse(line.slice(6));
        this.eventQueue.push({ ...this.currentEvent, data });
        this.flushQueue();
      }
    }
  }
  
  flushQueue() {
    // 按 step 排序
    this.eventQueue.sort((a, b) => 
      (a.data.step || 0) - (b.data.step || 0)
    );
    
    // 去重
    const seen = new Set();
    this.eventQueue = this.eventQueue.filter(e => {
      const key = `${e.type}-${e.data.step}-${e.data.tool || ''}`;
      if (seen.has(key)) return false;
      seen.add(key);
      return true;
    });
    
    // 依次处理
    while (this.eventQueue.length > 0) {
      const event = this.eventQueue.shift();
      this.onEvent(event);
    }
  }
}
```

八、风险与应对
================================================================================

风险 1：LLM 规划质量不稳定
--------------------------------------------------------------------------------
表现：任务拆解不合理，工具选择错误，陷入死循环
应对：
  ✓ 优化提示词，添加领域特定的 few-shot 示例
  ✓ 实现工具选择置信度评估（低于阈值则询问用户）
  ✓ 添加死循环检测（相同步骤重复 3 次则终止）
  ✓ 实现人工审核模式（关键任务需用户确认后再执行）

--------------------------------------------------------------------------------

风险 2：MCP 工具调用失败率高
--------------------------------------------------------------------------------
表现：API 限流、超时、返回错误、连接断开
应对：
  ✓ 实现指数退避重试机制（1s/2s/4s/8s）
  ✓ 实现分级超时控制（连接 10s/执行 60s）
  ✓ 实现降级策略（搜索失败则用缓存/备用 API）
  ✓ 添加健康检查（定期 ping MCP Server）

--------------------------------------------------------------------------------

风险 3：执行时间过长用户流失
--------------------------------------------------------------------------------
表现：复杂任务耗时>1 分钟，用户失去耐心
应对：
  ✓ 设置最大执行步数（默认 15 步，可配置）
  ✓ 支持并行工具调用（独立工具同时执行）
  ✓ 实现用户可打断机制（随时点击停止）
  ✓ 优化进度展示（让用户感知系统在工作）

--------------------------------------------------------------------------------

风险 4：前端性能瓶颈
--------------------------------------------------------------------------------
表现：长对话场景下滚动卡顿，内存泄漏
应对：
  ✓ 实现虚拟滚动（只渲染可见区域）
  ✓ 实现组件懒加载（按需加载重型组件）
  ✓ 优化状态更新频率（防抖/节流）
  ✓ 定期清理过期执行历史（保留最近 50 条）

--------------------------------------------------------------------------------

风险 5：MCP SDK 兼容性问题
--------------------------------------------------------------------------------
表现：不同版本 MCP SDK API 不一致，连接失败
应对：
  ✓ 锁定 MCP SDK 版本（requirements.txt 指定版本号）
  ✓ 实现适配器模式屏蔽版本差异
  ✓ 提供纯 HTTP 降级方案（不依赖 SDK）
  ✓ 添加详细的错误日志便于排查

十、总结
================================================================================

核心改造目标：
  从"场景驱动的专用系统"改造为"能力驱动的通用系统"

核心改造内容：
  1. 智能体：多智能体并行 → 单通用智能体 ReAct 循环
  2. 规划：硬编码规则 → LLM 动态规划 + 反思调整
  3. 工具：模拟调用 → 真实 MCP 工具 + 热插拔
  4. 执行：固定流程 → 动态适应 + 错误恢复
  5. 前端：一次性输出 → 流式渐进展示 + 状态可恢复

预期效果：
  ✓ 一个智能体，适应无限场景
  ✓ 新能力 = 新工具（无需改代码）
  ✓ 新场景 = 新规划（LLM 自动适配）
  ✓ 用户体验：全程透明，等待感知缩短 50%+
  ✓ 系统可维护性：代码量减少 30%，扩展性提升 10 倍

一句话总结：
  真正的通用智能体，不是"什么都会的超人"，而是"会用工具的普通人"。
  它不需要预设所有场景的解决方案，只需要理解目标、选择工具、达成结果。
  这次改造不是简单的代码重构，而是架构范式的根本转变。
